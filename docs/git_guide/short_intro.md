Сделаем компактную «боевую памятку» под твой уровень.

---

## 0. Общая схема работы

1. Есть **основная ветка**: `main` (или `master`).
2. Под каждую задачу делаешь **отдельную feature-ветку**: `feat/hop-parsing`, `feat/greedy`, и т.п.
3. Ветку → коммиты → пуш → PR → merge в `main`.

---

## 1. Создание репозитория

### Вариант А: сначала локально, потом GitHub

1. В корне проекта:
   - `git init`
2. Добавить файлы:
   - `git add .`
3. Первый коммит:
   - `git commit -m "Initial commit"`
4. На GitHub создаёшь **пустой репозиторий** (без авто-README).
5. Связать локальный и удалённый:
   - `git remote add origin <SSH-или-HTTPS-URL>`
6. Запушить основную ветку:
   - `git branch -M main`
   - `git push -u origin main`

### Вариант B: сначала GitHub, потом локально

1. Создаёшь репозиторий на GitHub (с пустым README или без разницы).
2. Локально:
   - `git clone <URL>`
3. Работаешь уже в склонированной папке.

Оба варианта норм. Для тебя проще **B**: сделал репо на GitHub → `git clone` → всё.

---

## 2. Feature-ветка: как работать

Ты уже в папке проекта, `main` обновлён.

1. Обновить `main` (если есть удалённый):
   - `git switch main`
   - `git pull`
2. Создать новую ветку и перейти в неё:
   - `git switch -c feat/hop-parsing`
3. Работаешь, изменяешь файлы.
4. Проверяешь, что изменилось:
   - `git status`
   - `git diff` (по желанию)
5. Добавляешь изменения:
   - `git add .`
     или выборочно: `git add path/to/file`.
6. Делаешь коммит:
   - `git commit -m "Implement input parsing for HOP task"`
7. Первый пуш новой ветки:
   - `git push -u origin feat/hop-parsing`

После этого ветка появится на GitHub.

---

## 3. Pull Request и merge

Дальше удобнее делать **на сайте GitHub**:

1. Открываешь репозиторий на GitHub.
2. Он обычно покажет баннер «Compare & pull request» для новой ветки → жмёшь.
3. В PR:
   - base: `main`
   - compare: `feat/hop-parsing`
   - пишешь нормальное описание.
4. Создаёшь PR.
5. Просматриваешь diff, если всё ок — жмёшь **Merge** (обычно `Merge pull request` → `Confirm`).
6. После merge можно удалить ветку:
   - на GitHub: кнопка `Delete branch`;
   - локально:
     - `git switch main`
     - `git pull`
     - `git branch -d feat/hop-parsing`

---

## 4. Где что делать: локально vs сайт

- **Локально**:
  - редактирование файлов;
  - `add/commit`;
  - создание/переключение веток;
  - первые пуши.

- **На GitHub (в браузере)**:
  - создание репозитория (проще так);
  - создание PR;
  - merge PR;
  - просмотр истории/диффов.

Это нормальный рабочий минимум для твоего текущего уровня.

Если захочешь, потом можем добавить слой «как нормально называть коммиты и ветки», но это уже косметика.
